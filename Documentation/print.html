Micro Core

MicroCore

This repository contains the code for the Micro Core Library, this project defined Assertion, File wrapper, math types definitions and functions, platform's specific wrapper function, Multi-threading work management and Yaml file support.

Build System

MicroCore as part of the Micro project use the Premake5 build system. Premake5 instance is present in this repository under Build/Premake5.

Build files for Premake5 are inside Build folder under :

FileUsage
Build/Build.luaDefine global solution build.
Build/Build-MicroCore.luaDefine MicroCore solution build.
Build/Build-Thirdparty.luaDefine Thirdparty solution build.

Windows

For Windows build you need at least visual studio 2022 community edition or visual studio C++.

To build the project files simply run Setup-Windows.bat

Linux

caution

Linux isn't implemented yet.

To build the project files simply run Setup-Linux.sh

Assert

This section of the documentation present how assertion is defined inside MicroCore library.

When MICRO_DEBUG is declared the micro_assert macro act as same as native assert function, when ins't declared the micro_assert macro is defined as empty so no assertion is evaluated. To enable MICRO_DEBUG you need to define it before your include of the library.

#define MICRO_DEBUG
#include "xxxxx/MicroCore.h"

Macro

Sunytax

micro_assert( CHECK, FORMAT, ARGS );

Parameters

NameDescription
CHECKAssertion condition.
FORMATString format as spdlog format.
ARGSArguments list to populate the format.

Example

As mention if the `MICRO_DEBUG is declared, the follow assert will check if a is equal to b and if the condition fail print the message "A is not equal to B !" and emit debugger break point.

micro_assert( a == b, "A is not equal to B !" );

File

Syntax

enum class MicroFileAccessors : uint32_t {

	MFA_NONE  = 0,
	MFA_BINARY = 1 << 0,
	MFA_READ   = 1 << 1,
	MFA_WRITE  = 1 << 2,
	MFA_APPEND = 1 << 3,

	MFA_BINARY_READ   = MFA_BINARY | MFA_READ,
	MFA_BINARY_WRITE  = MFA_BINARY | MFA_WRITE,
	MFA_BINARY_APPEND = MFA_BINARY | MFA_APPEND

};

Values

NameDescription
MFA_NONEThe none value is used to defined no accessor, represent no file access.
MFA_BINARYFlag to define that the file is in binary format.
MFA_READFlag to define that the file is used for read operation.
MFA_WRITEFlag to define that the file is used for write operation.
MFA_APPENDWhen used, file is open as append.
MFA_BINARY_READWhen used, file is open for read binary data.
MFA_BINARY_WRITEWhen used, file is open for write binary data.
MFA_BINARY_APPENDWhen used, file is open as apend for read and write binary data.

File : Example

Write Text

// Create file instance.
MicroFile file = MicroFile( );

// Open file for writing.
if ( MicroFile.Open( "xxxxx.txt", MicroFileAccessors::MFA_WRITE ) ){
	micro_string text = "Hello !\n";

	// Write.
	file.Write( (uint32_t)strlen( text ), text );

	// Close file.
	file.Close( );
}

Write Binary

// Create file instance.
MicroFile file = MicroFile( );

// Open file for writing.
if ( MicroFile.Open( "xxxxx.bin", MicroFileAccessors::MFA_BINARY_WRITE ) ){
	int a = 10;

	// Write.
	file.Write( (uint32_t)sizeof( int ), &a );

	// Template write.
	file.Write( a );

	// Close file.
	file.Close( );
}

Read Text

// Create file instance.
MicroFile file = MicroFile( );

// Open file for writing.
if ( MicroFile.Open( "xxxxx.txt", MicroFileAccessors::MFA_READ ) ){
	char text[ 10 ];

	// Write
	file.Read( 10 * (uint32_t)sizeof( char ), text );

	// Close file.
	file.Close( );
}

Read Binary

// Create work manager instance.
MicroWorkManager work_manager = MicroWorkManager( );

// Setup work manager.
if ( work_manager.Create( ) ) {
	// Create work instance to push.
	// Use simple execution lambda to print thread work id.
	MicroWork work = MicroWork(
		[]( MicroWorkManager* work_manager, float* storage ) {
			printf( "%u\n", std::this_thread::get_id( ) );

	// Destroy work manager.
	work_manager.Destroy( );
}

Yaml : Custom Nodes

The following code snippet defined how the the library defined yaml support for micro_upoint custom type. For more details for extending supported types see the official yaml-cpp documentation.

template<>
struct convert<micro_upoint> {
    static Node encode( const micro_upoint& rhs ) {
        Node node;

        node.push_back( rhs.x );
        node.push_back( rhs.y );

        return node;
    }

    static bool decode( const Node& node, micro_upoint& rhs ) {
        if ( !node.IsSequence( ) || node.size( ) != 2 )
            return false;

        rhs.x = node[ 0 ].as<uint32_t>( );
        rhs.y = node[ 1 ].as<uint32_t>( );

        return true;
    }

Yaml Reader : Example

// Create reader instance.
MicroYamlReader reader = MicroYamlReader( );

// Try to open file to read.
if ( reader.Open( "xxxxx.yaml" ) ) {
	// Read the pair from the file :
	// KeyName : 'Value'

	// Close the file.
	reader.Close( );
}
